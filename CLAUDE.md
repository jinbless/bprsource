# 고용노동부 예산사업-정보시스템 매핑 프로젝트

## 프로젝트 개요
고용노동부 산하 197건 정보시스템의 예산사업 매핑 현황을 분석하고, 시스템 성격에 따른 카테고리 분류를 수행하는 프로젝트.

## 주요 파일
- `예산사업_시스템_매핑.csv` — **핵심 데이터 파일**. 197건 시스템 × 17개 컬럼
- `pdf/` — 세부사업별 상세 설명 MD 파일 (약 70개). 파일명 형식: `{번호}_고용노동부_{세부사업명}.md`
- `apply_corrections.py` — 카테고리 수정 스크립트 (29건 수정 이력 포함)

## CSV 컬럼 구조
### 원본 컬럼 (10개)
기관명, 시스템명, 등급, DR유무, 이중화유무, 예산사업No, 세부사업명, 단위사업명, 국회확정금액, 매칭방법

### 추가된 카테고리 컬럼 (7개)
| 컬럼명 | 값 목록 | 비고 |
|--------|---------|------|
| 서비스대상 | 대국민, 내부, 혼합 | 최종 사용자 기준 |
| 업무도메인 | 고용서비스, 사회보험, 산업안전보건, 직업훈련·교육, 자격·평가, 근로복지, 노사관계, 외국인고용, 장애인고용, 행정지원, 정보공개·홍보 | 11개 값 |
| 시스템유형 | 홈페이지/포털, 업무처리시스템, 데이터/분석, 교육/학습(LMS), 내부인프라, 상담/콜센터, 금융/기금, 의료정보 | 8개 값 |
| 기관유형 | 본부, 위원회, 공단, 정보원, 교육기관, 재단/진흥원, 공제회, 기타공공 | 기관명 기반 |
| 금융관련 | Y, N | 보험료·기금·연금·대부 관련 여부 |
| 개인정보수준 | 고, 중, 저 | 민감정보 처리 수준 추정 |
| 서비스채널 | 웹, 앱, 콜, 복합 | 시스템명 기반 추정 |

## 세부사업 ↔ PDF 연계
- CSV의 `세부사업명` 컬럼이 pdf/ 폴더 MD 파일의 key
- 24개 고유 세부사업명이 존재
- MD 파일에서 "사업목적 내용", "사업개요" 섹션이 사업 성격 파악에 핵심

## 작업 이력
1. 7개 카테고리 컬럼 설계 및 CSV에 추가 (197건)
2. PDF 세부사업 분석을 통해 카테고리 검증 → 29건 수정 반영
3. 수정 유형: 업무도메인 변경(18건), 서비스대상 변경(6건), 시스템유형 변경(6건), 개인정보수준 변경(1건)

## 코딩 규칙
- CSV 인코딩: utf-8-sig (BOM 포함)
- Python 실행: `py -X utf8` (Windows 환경)
